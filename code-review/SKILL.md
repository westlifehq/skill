# Code Review Skill

当被要求进行 Code Review 时，请遵循以下准则：

1. **逻辑正确性**：检查是否存在潜在的逻辑错误、边界情况考虑不全或空指针异常等问题。
2. **性能与安全**：识别可能导致性能瓶颈（如不必要的循环嵌套、多次 DB 查询）或安全漏洞（如 SQL 注入、XSS）的代码。
3. **代码整洁度**：检查变量命名是否清晰、函数是否遵循单一职责原则、代码是否有过多重复。
4. **输出格式**：
   - 给出简明扼要的总结。
   - 使用 Markdown 列表列出具体发现的问题，并**直接提供修改后的代码片段**。
   - 不要给出过于泛泛的建议，要具体到行。
